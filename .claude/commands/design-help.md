아래 내용을 터미널에 표시합니다:

```
설계 작업 순서

1단계: UI/UX 설계
/design-uiux
- UI/UX설계가이드를 준용하여 작성

2단계: 프로토타입 작성
/design-prototype
- 프로토타입작성가이드를 준용하여 작성

3단계: 프로토타입 테스트
/design-test-prototype
- 프로토타입을 테스트

4단계: 프로토타입 오류수정 (필요시)
/design-fix-prototype
- [오류내용] 섹션에 오류 내용 제공 필요

5단계: 프로토타입 개선 (필요시)
/design-improve-prototype
- [개선내용] 섹션에 개선할 내용 제공 필요

6단계: 유저스토리 품질 높이기
/design-improve-userstory
- 프로토타입 분석 후 유저스토리 업데이트

7단계: 설계서 업데이트
/design-update-uiux
- UI/UX설계서와 스타일가이드 수정

8단계: 클라우드 아키텍처 패턴 선정
/design-pattern
- 클라우드아키텍처패턴선정가이드 준용

9단계: 논리아키텍처 설계
/design-logical
- 공통설계원칙과 논리아키텍처설계가이드 준용

10단계: 외부 시퀀스 설계
/design-seq-outer
- 외부시퀀스설계가이드 준용

11단계: 내부 시퀀스 설계
/design-seq-inner
- 내부시퀀스설계가이드 준용

12단계: API 설계
/design-api
- API설계가이드 준용

13단계: 클래스 설계
/design-class
- [클래스설계 정보] 섹션에 패키지 그룹, 설계 아키텍처 패턴 제공 필요

14단계: 데이터 설계
/design-data
- 데이터설계가이드 준용

15단계: High Level 아키텍처 정의서 작성
/design-high-level
- CLOUD 정보 제공 필요 (Azure, AWS, Google)

16단계: 물리 아키텍처 설계
/design-physical
- CLOUD 정보 제공 필요 (Azure, AWS, Google)

17단계: 프론트엔드 설계
/design-front
- [백엔드시스템] 섹션에 시스템 정보 제공 필요
```
