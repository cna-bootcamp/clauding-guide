# 아이디어 발상

## 목적

문제 가설과 방향성을 기반으로 창의적인 솔루션 아이디어를 생성하고 수렴합니다.

## 사용 시점

- 문제 가설과 방향성 정의가 완료된 후
- 솔루션 선정 전
- 여러 접근 방식을 탐색해야 할 때
- 사용자가 "아이디어", "브레인스토밍", "솔루션 발상"을 언급할 때

## 필수 입력

- 타겟 고객 정의: `define/고객분석.md`
- 문제 가설: `define/문제가설.md`
- 고객 경험 조사 결과:
  - `define/체험결과.md`
  - `define/관찰결과.md`
  - `define/고객경험인터뷰결과.md`
- 문제 검증 결과:
  - `define/문제검증인터뷰결과.md`
  - `define/문제검증인터뷰결과취합.md`
- 전략적 방향: `think/문제해결방향성.md`

## 브레인스토밍 원칙

- **양이 질을 낳는다**: 최대한 많은 아이디어 생성
- **판단 유예**: 모든 아이디어 수용
- **자유로운 사고**: 창의적이고 대담한 아이디어 장려
- **사용자 중심**: 각자의 전문성을 내려놓고 사용자 입장에서 아이디어 도출

## 아이디어 발상 기법

아이디어를 체계적으로 발산하기 위한 두 가지 핵심 기법을 활용하세요.

### 1️⃣ SCAMPER 기법

기존 아이디어나 솔루션을 7가지 관점에서 변형하여 새로운 아이디어를 창출합니다.

#### **S - Substitute (대체)**
무엇을 다른 것으로 대체할 수 있는가?
- 재료, 자원, 프로세스, 사람, 장소를 바꾸면?
- 다른 기술, 다른 접근방식을 사용하면?

**질문 예시**:
- 사람이 하던 일을 AI로 대체하면?
- 오프라인 경험을 온라인으로 대체하면?
- 유료 서비스를 무료로 대체하면 어떻게 수익화?

#### **C - Combine (결합)**
무엇을 결합할 수 있는가?
- 기능, 아이디어, 목적을 합치면?
- 여러 서비스를 하나로 통합하면?

**질문 예시**:
- 우리 서비스 + 다른 산업의 서비스 = ?
- A 기능 + B 기능 = 새로운 가치?
- 온라인 + 오프라인 경험을 결합하면?

#### **A - Adapt (적용)**
다른 분야에서 무엇을 가져올 수 있는가?
- 유사한 문제를 다른 산업에서는 어떻게 해결했나?
- 과거의 성공 사례를 현재에 적용하면?

**질문 예시**:
- 게임의 보상 시스템을 우리 서비스에 적용하면?
- Netflix의 추천 알고리즘을 우리 도메인에 적용하면?
- 병원의 예약 시스템을 다른 산업에 적용하면?

#### **M - Modify/Magnify/Minify (수정/확대/축소)**
무엇을 변경, 확대, 축소할 수 있는가?
- 크기, 모양, 색상, 의미를 바꾸면?
- 기능을 더 강력하게 또는 더 단순하게 만들면?

**질문 예시**:
- 기능을 10배 확대하면? 1/10로 축소하면?
- 타겟 고객을 10배 넓히면? 특정 니치로 좁히면?
- 사용 시간을 1시간에서 1분으로 줄이면?

#### **P - Put to other uses (다른 용도)**
다른 방식으로 사용할 수 있는가?
- 원래 목적과 다른 용도로 활용하면?
- 다른 시장, 다른 고객에게 제공하면?

**질문 예시**:
- B2C 제품을 B2B로 전환하면?
- 부수적 기능을 주 기능으로 만들면?
- 부산물을 새로운 제품으로 만들면?

#### **E - Eliminate (제거)**
무엇을 제거할 수 있는가?
- 불필요한 기능, 단계, 규칙을 없애면?
- 최소한의 핵심만 남기면?

**질문 예시**:
- 회원가입 없이 서비스를 이용하게 하면?
- 복잡한 설정 과정을 모두 제거하면?
- 앱 다운로드 없이 웹만으로 제공하면?

#### **R - Reverse/Rearrange (역전/재배열)**
순서를 바꾸거나 역으로 생각하면?
- 프로세스의 순서를 바꾸면?
- 원인과 결과를 뒤집으면?

**질문 예시**:
- 고객이 먼저 결제하는 대신 나중에 결제하면?
- 공급자가 아닌 소비자가 가격을 정하면?
- 플랫폼이 아닌 사용자가 수익을 얻으면?

### 2️⃣ Steal & Synthesize (훔쳐서 재합성)

다른 산업과 분야의 성공 사례를 분석하고, 우리 문제에 맞게 재조합하는 기법입니다.

#### **Steal (훔치기)**

**1단계: 영감의 원천 찾기**
- 다른 산업의 성공 사례 조사
- 글로벌 트렌드 및 혁신 기업 분석
- 전혀 다른 도메인의 문제 해결 방식 탐색

**조사 영역**:
- 🎮 **게임**: 사용자 참여, 보상 체계, 레벨 시스템
- 🏥 **헬스케어**: 개인화, 데이터 추적, 습관 형성
- 🏦 **금융**: 신뢰 구축, 보안, 간편 결제
- 🛒 **이커머스**: 추천 시스템, 구매 전환, 배송 혁신
- 🎓 **교육**: 학습 경로, 피드백, 성취 관리
- 🚗 **모빌리티**: 공유 경제, 실시간 매칭, 위치 기반 서비스
- 🎬 **엔터테인먼트**: 개인화 콘텐츠, 구독 모델, 몰입 경험
- 🏭 **제조업**: 자동화, 품질 관리, 공급망 최적화

**질문**:
- Uber는 택시 산업의 어떤 문제를 어떻게 해결했나?
- Netflix는 어떻게 사용자를 지속적으로 참여시키나?
- Duolingo는 어떻게 학습을 게임화했나?
- Airbnb는 어떻게 신뢰를 구축했나?

**2단계: 핵심 메커니즘 추출**
성공 사례의 본질적인 작동 원리 파악:
- **무엇이** 문제였나?
- **어떻게** 해결했나?
- **왜** 효과적이었나?
- **핵심 메커니즘**은 무엇인가?

#### **Synthesize (재합성)**

**3단계: 우리 문제에 적용**
추출한 메커니즘을 우리 도메인에 맞게 변형:

**적용 프로세스**:
1. **유사점 찾기**: 우리 문제와 성공 사례의 공통점은?
2. **차이점 인식**: 우리만의 제약사항과 특수성은?
3. **변형 설계**: 핵심 메커니즘을 우리 맥락에 맞게 변형
4. **검증**: 우리 사용자에게 실제로 효과적일까?

**재합성 예시**:

| 원천 사례 | 핵심 메커니즘 | 우리 도메인 적용 |
|----------|-------------|----------------|
| 게임 레벨 시스템 | 점진적 난이도 상승, 성취감 | 학습 앱의 단계별 커리큘럼 |
| Spotify 일일 믹스 | AI 기반 개인화 추천 | 뉴스 앱의 맞춤 콘텐츠 |
| 항공 마일리지 | 충성도 보상 프로그램 | 카페 멤버십 포인트 |
| 우버 실시간 추적 | 투명성과 안심 제공 | 배달 앱의 배송 추적 |
| 넷플릭스 자동재생 | 마찰 최소화, 지속 시청 | 교육 앱의 자동 다음 강의 |

**4단계: 여러 아이디어 조합**
2-3개의 다른 산업 아이디어를 결합하여 혁신적 솔루션 창출:

**조합 예시**:
- **Tinder (스와이프) + Airbnb (신뢰) + Uber (실시간 매칭)** → 새로운 서비스
- **게임 (보상) + 금융 (저축) + SNS (공유)** → 재미있는 저축 앱
- **넷플릭스 (추천) + 교육 (학습) + 게임 (레벨)** → 개인화 학습 플랫폼

#### **활용 팁**

✅ **좋은 Steal & Synthesize**:
- 핵심 메커니즘을 정확히 이해
- 우리 맥락에 맞게 창의적으로 변형
- 여러 아이디어를 조합하여 독창성 확보

❌ **피해야 할 것**:
- 표면적인 모방 (UI만 복사)
- 맥락 무시한 직접 복제
- 메커니즘 이해 없는 적용

## 1단계: 솔루션 탐색

모든 팀원들이 각자의 아이디어를 발산합니다.

### 아이디어 구분

- **Big Idea**: 인력과 비용은 많이 드나 실현되면 비즈니스 기여도가 높은 아이디어
- **Little Win Idea**: 실현가능성이 높은 아이디어
- **Crazy Idea**: 절대 불가능할 것 같지만 실현되면 비즈니스 기여도가 가장 높은 아이디어

### 요청사항

- '문제해결 방향성'에 맞게 **Big Idea 3개**, **Little Win Idea 2개**, **Crazy Idea 1개**를 각자 도출
- **(중요)** 각자의 전문성을 내려놓고 사용자 입장에서 솔루션 방향성에 맞는 아이디어를 도출
- Sequential MCP 이용

### 아이디어 발상 가이드라인

#### **기본 원칙**
- 각 아이디어는 구체적이고 실행 가능해야 함
- 고객 문제 해결에 직접 집중
- 여러 관점 고려
- 초기 생성 시 자기 검열 금지
- 비전통적인 아이디어도 모두 문서화

#### **발상 기법 활용**

**SCAMPER 활용**:
- Big Idea 도출 시: **Adapt**, **Combine**, **Reverse** 관점 적극 활용
- Little Win Idea 도출 시: **Eliminate**, **Modify** 관점으로 단순화
- Crazy Idea 도출 시: **Magnify**, **Put to other uses** 관점으로 극단화

**Steal & Synthesize 활용**:
- 다른 산업의 성공 사례를 최소 3개 이상 조사
- 핵심 메커니즘을 우리 도메인에 맞게 변형
- 2-3개 아이디어를 조합하여 독창적인 솔루션 창출

**발상 프로세스 예시**:
1. **Steal**: 타 산업 사례 3개 조사 → 핵심 메커니즘 추출
2. **SCAMPER**: 추출한 메커니즘에 SCAMPER 7가지 질문 적용
3. **Synthesize**: 여러 아이디어를 조합하여 Big/Little Win/Crazy 아이디어 도출

### 결과 형식: 마크다운 표
각 팀원의 아이디어를 작성   
```markdown
# 솔루션 탐색

| 팀원 |  아이디어 구분 | 아이디어 제목 | 아이디어 설명 |
|--------------|--------------|--------------|
| 갑빠 | Big Idea | {아이디어 제목} | {상세 설명} |
| 갑빠 | Big Idea | {아이디어 제목} | {상세 설명} |
| 갑빠 | Big Idea | {아이디어 제목} | {상세 설명} |
| 갑빠 | Little Win Idea | {아이디어 제목} | {상세 설명} |
| 갑빠 | Little Win Idea | {아이디어 제목} | {상세 설명} |
| 갑빠 | Crazy Idea | {아이디어 제목} | {상세 설명} |
```

### 결과 파일

`think/솔루션탐색.md`

## 2단계: 솔루션 수렴

각자 발산한 아이디어를 수렴하여 솔루션 후보를 정리합니다.

### 요청사항

- **아이디어 유사도 평가표 작성**: 유사도 0.7 이상일 때만 아이디어를 합침
  - **기능 유사도** (가중치 70%): 뭘 해주는 서비스인가? 해결하려는 문제와 제공 가치의 유사성
  - **사용자 경험 유사도** (가중치 30%): 어떻게 사용하는 서비스인가? 서비스 이용 방식과 인터페이스의 유사성
- **정리된 각 아이디어의 구성**:
  - 첫 줄: 아이디어 구분: 아이디어 제목
  - 둘째 줄: 아이디어 설명 (최대한 자세히 표현)
- Sequential MCP 이용

### 유사도 평가 예시

```markdown
## 유사도 평가표

| 아이디어 A | 아이디어 B | 기능 유사도 | 경험 유사도 | 종합 유사도 | 합치기 여부 |
|-----------|-----------|------------|------------|-------------|-----------|
| Big Idea 1 | Big Idea 2 | 0.8 | 0.6 | 0.74 | ✅ 합침 |
| Little Win 1 | Crazy Idea | 0.4 | 0.3 | 0.37 | ❌ 별도 유지 |
```

### 결과 형식: 문단형식

```markdown
# 솔루션 후보

## Big Idea: {아이디어 제목}
{아이디어 설명을 최대한 자세히 표현}

## Little Win Idea: {아이디어 제목}
{아이디어 설명을 최대한 자세히 표현}

## Crazy Idea: {아이디어 제목}
{아이디어 설명을 최대한 자세히 표현}
```

## 도구 활용

### Sequential MCP 사용
복잡한 아이디어 발상과 조합, 유사도 평가가 필요할 때 Sequential MCP를 활용하여 체계적으로 창의적 솔루션을 도출하세요.

## 주의사항

- **아이디어 발상 기법**:
  - SCAMPER 7가지 관점을 체계적으로 적용
  - Steal & Synthesize로 타 산업 사례 최소 3개 분석
  - 기법을 기계적으로 적용하지 말고 창의적으로 활용
  - 여러 기법을 조합하여 독창적인 아이디어 도출

- **1단계 (솔루션 탐색)**:
  - 각자의 전문성을 내려놓고 사용자 관점에서 생각
  - Big Idea 3개, Little Win Idea 2개, Crazy Idea 1개 반드시 도출
  - 초기 생성 시 자기 검열 금지
  - SCAMPER와 Steal & Synthesize 기법 적극 활용

- **2단계 (솔루션 수렴)**:
  - 유사도 0.7 이상일 때만 합치기
  - 기능 유사도 70%, 경험 유사도 30% 가중치 적용
  - 아이디어 설명은 최대한 자세하게 작성

## 다음 단계

아이디어 발상 완료 후:
1. **결과 파일 팀원들과 검토 및 수정**
2. 솔루션 선정 (우선순위 평가)
3. MVP 범위 정의
4. 비즈니스 모델 설계
